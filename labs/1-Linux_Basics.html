Manipulating files and directories
  1. Use the ls command after each step to confirm that you did it correctly, and use the pwd command to see which directory you are in.
  2. In your Linux home directory, make a new directory called transcriptomics.
  3. Change to the transcriptomics directory, and confirm that you are in the transcriptomics directory.
  4. Make a new directory called linux, change to the directory, and confirm that you are in the linux directory.
  5. Make an empty text file called empty.txt with the command touch empty.txt.
  6. Copy the file empty.txt to empty.txt.bak.
  7. Rename the file empty.txt.bak to empty2.txt.
  8. Create a subdirectory called sub1.
  9. Move the file empty2.txt into the sub1 directory.
  10. Create another subdirectory called sub2.
  11. Move the file empty2.txt in the sub1 directory into the sub2 directory.
  12. Move the file empty2.txt in the sub2 directory back into the linux directory, and change the file name to empty3.txt. Do the moving and renaming at the same time, with one command.
  13. Remove the file empty3.txt.
  14. Remove the sub1 and sub2 directories.
  15. You should now have only the file empty.txt in your linux directory. Confirm that this is true.
Viewing files
  1. Copy the file /export/data/657.001/text/exons.fasta into the linux directory.
  2. Display the file exons.fasta on your terminal using the cat command.
  3. Display the file exons.fasta on your terminal using the less command.
  4. Navigate through the file with the less command. At the : at the bottom of the terminal, try these keys: SPACE and b, d and u, and ENTER and y.
  5. Type / at the : prompt while using less, to enter the search mode. Search for the word length in the file exons.fasta.
Editing files
  1. Use the command gedit exons.fasta to edit the file exons.fasta. If you only have a terminal connection to the Linux server, you can do nano exons.fasta to use another text editor.
  2. Remove everything in the file except the first exon sequence. To do this, search for the second sequence header (i.e. the > character) and remove everything from the line to the end of the file.
  3. Save the changed file as a new file called exon1.fasta.
  4. Confirm that you now have both exons.fasta and exon1.fasta.
Finding help
  1. Use the command man ls to find the option to ls to list all files, including hidden ones. Try it in your home directory.
  2. What is the option to ls to list all files in all subdirectories? Try it in your home directory.
  3. What is the option to wc to show only the number of lines?
  4. What is the option to sort to sort in numeric order, instead of the default alphabetic order?
  5. What is the option to grep to select the lines not containing the given string?
Redirection and piping
  1. Move to your linux directory (if not already there). Using wildcard characters (*, ?, or []), copy only those files that start with seq, from the directory /export/data/657.001/text to your linux directory, using this command: 

    cp /export/data/657.001/text/seq* .

  2. Use ls and wildcards to list all files with a name that starts with seq and contains exactly 5 characters.
  3. Redirect (use >) the out of the previous ls command to a new file called seqfiles.txt.
  4. Collect the contents of the files in seqfiles.txt into a single file. Use the cat command and output redirection.
  5. What is a single Linux command to count how many files you have in your current directory? You need to use piping.
